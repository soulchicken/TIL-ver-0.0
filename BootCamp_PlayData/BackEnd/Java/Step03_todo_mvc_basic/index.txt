Step03 : 관심사의 분리
1. 읽어들인 데이터를 parsing해주는 코드를 별도의 클래스인 TodoTSVParser.java로 분리
2. 분리한 클래스가 위치한 패키지명은 data
	- 데이터 접근 역할에 해당하는 부분이기 때문
3. App.java에서 조회에 해당하는 코드도 별도의 메서드로 분리

개선된 부분
1. 데이터를 파싱하는 부분을 별도의 클래스로 관심사(역할)을 분리.
2. 관리가 편해짐

한계
1. 데이터 파싱부분은 분리했지만, 여전히 App.java에서 출력하는 부분(view)
2. 연산처리(id값에 다라 특정 Todo 데이터 조회)등을 전부 수행하는 중
